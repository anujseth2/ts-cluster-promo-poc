Engine ② Discover all issues
ThoughtSpot's validate stops at the first error per table. So we loop on a throwaway copy — validate, classify, neutralise, re-validate — until the copy is clean. The real bundle is never touched. Findings are unioned across passes, so you see the complete list up front, not one at a time.
repeats until clean — or a pass can't neutralise anything
clean ✓ → resolve no_progress → manual request_failed → keep prior
③ Resolve findings — route each to a column-level action Each discovered blocker maps to a reviewer choice. Dropping a column cascades its whole blast radius automatically, so imports never fail on dangling references.
missing_in_target_warehouse
Column not in the CDW (err 14536). Tick / Select all → drop, or add to warehouse.
type_mismatch
Declared type ≠ warehouse type. Retype, align warehouse, or drop.
invalid_formula_ids
Orphaned formula. Drop by name — takes its surfacing column too.
drop_blocked_by_dependents
Target column has dependents. Preserve it, or remove deps first.
viz_error
Viz won't load. Drop the viz; its layout tile is pruned with it.
other  ·  opaque
"Schema validation failed", no name. Isolate per-file to pin the culprit.
